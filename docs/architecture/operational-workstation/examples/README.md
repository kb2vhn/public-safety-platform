# Operational Workstation Examples

> These files are non-production examples.
>
> They contain placeholder values and incomplete deployment-specific decisions. They must not be deployed unchanged.

The examples demonstrate the intended separation between:

- Approved configuration.
- observed state.
- trust evidence.
- module profiles.
- release artifacts.
- network communications.
- workspace bindings.
- Operational Fault Episodes.
- systemd supervision and socket activation.

Formal schemas and validators should be added when implementation begins.
