Public Safety Platform Foundation SQL Test Framework
=====================================================

Purpose
-------

This directory is intentionally self-contained beneath:

    sql/test-framework/

Do not copy its files into the repository root or move its SQL files into the
live migration tree.

The framework reads the deployable Foundation migrations from:

    sql/schema/

It keeps all test-only SQL, runner files, logs, and summaries beneath:

    sql/test-framework/

Directory relationship
----------------------

Live Foundation SQL:

    sql/schema/manifests/foundation.manifest
    sql/schema/migrations/foundation/

Test framework:

    sql/test-framework/Makefile
    sql/test-framework/sql/schema/scripts/test_foundation.sh
    sql/test-framework/sql/tests/
    sql/test-framework/sql/test-results/

The test-only sql_test schema is created only inside a disposable test
database. It is not part of the live Foundation migration manifest.

Prerequisites
-------------

Required software:

    PostgreSQL 18 or newer
    Bash
    GNU core utilities
    psql
    createdb
    dropdb
    sha256sum
    make

The PostgreSQL role used to run the framework must be allowed to:

    connect to the maintenance database
    create databases
    drop the disposable test database
    create the objects required by the Foundation migrations

Initial setup
-------------

From the repository root:

    chmod +x sql/test-framework/sql/schema/scripts/test_foundation.sh

Run the framework
-----------------

Run from the self-contained test-framework directory:

    cd sql/test-framework
    make test-sql

The runner will:

    1. Resolve the repository root from its current installed location.
    2. Read the live manifest from sql/schema/manifests/foundation.manifest.
    3. Create a uniquely named disposable PostgreSQL database.
    4. Apply every live Foundation migration in manifest order.
    5. Calculate the SHA-256 digest of each migration file.
    6. Install the test-only sql_test assertion framework.
    7. Run every test listed in foundation-tests.manifest.
    8. Write a full timestamped log and a compact text summary.
    9. Drop the database after success unless instructed to retain it.
   10. Retain a failed database by default for investigation.

Other run modes
---------------

Keep a successful test database:

    make test-sql-keep

Drop a failed database instead of retaining it:

    make test-sql-drop-failed

Run the shell script directly:

    ./sql/schema/scripts/test_foundation.sh

Show all runner options:

    ./sql/schema/scripts/test_foundation.sh --help

PostgreSQL connection settings
------------------------------

The runner uses normal libpq environment variables:

    PGHOST
    PGPORT
    PGUSER
    PGPASSWORD
    PGSSLMODE

Example:

    export PGHOST=127.0.0.1
    export PGPORT=5432
    export PGUSER=jwood

The maintenance database defaults to postgres. Override it with:

    export PGMAINTENANCE_DB=postgres

An optional test database name may be supplied with:

    export TEST_DATABASE_NAME=psp_foundation_test_manual

For safety, a supplied name must begin with:

    psp_foundation_test_

Output files
------------

Results are written beneath:

    sql/test-framework/sql/test-results/

Timestamped files:

    foundation_YYYYMMDD_HHMMSS_PID.log
    foundation_YYYYMMDD_HHMMSS_PID-summary.txt

Convenience links for the newest run:

    latest.log
    latest-summary.txt

The full log contains migration output, PostgreSQL errors, test notices,
warnings, and the final result inventory.

The summary contains:

    overall pass or fail status
    runner exit status
    assertion totals
    failed assertions
    warnings
    manifest and migration-registry totals

Database retention behavior
---------------------------

Successful run:

    The disposable database is dropped by default.

Failed run:

    The disposable database is retained by default so it can be inspected with
    psql or pgAdmin. The exact database name is printed in the log.

To inspect a retained database:

    psql --dbname=DATABASE_NAME

To remove it manually:

    dropdb --maintenance-db=postgres DATABASE_NAME

Review or upload the result
---------------------------

For a normal review, open or upload:

    sql/test-framework/sql/test-results/latest.log

For a compact overview, open or upload:

    sql/test-framework/sql/test-results/latest-summary.txt

Troubleshooting
---------------

Check shell syntax:

    bash -n sql/schema/scripts/test_foundation.sh

Run while retaining a successful database:

    KEEP_TEST_DB=1 make test-sql

Run and remove a failed database:

    DROP_TEST_DB_ON_FAILURE=1 make test-sql

Write results to another directory:

    ./sql/schema/scripts/test_foundation.sh \
        --results-dir /path/to/results

The framework must remain at sql/test-framework for its relative path
resolution to match this repository layout.
